ABOUT THE PROJECT

One application, multiple healthy servers.

I created this project to demonstrate load balancing, automatic scaling, health checks, instance replacement, private service access, and monitoring in a fault-tolerant AWS environment.

DESIGN CHOICES

What this deployment demonstrates

Multi-AZ availability

Instances run in separate Availability Zones so traffic can continue if one target fails.

Automatic recovery

Auto Scaling maintains capacity and replaces instances that fail EC2 or load balancer health checks.

Central monitoring

CloudWatch tracks load balancer, instance, and Auto Scaling metrics and supports operational alarms.
